# 템플릿 등록 방법 (AI 개발자용)

## 1. 템플릿 파일 위치

템플릿 파일은 **프로젝트 루트의 `templates/` 폴더**에 폴더 단위로 추가합니다.

```
templates/
  fashion/      ← 기존 템플릿
  sneaker/
  furniture/
  my-new-template/  ← 새 템플릿은 여기에 추가
```

> `src/` 폴더는 절대 건드리지 마세요. 템플릿 파일은 오직 `templates/` 에만 추가합니다.

---

## 2. 푸시 방법

```bash
git add templates/{템플릿명}/
git commit -m "feat: add {템플릿명} template"
git push origin main
```

---

## 3. 폴더 구조 규칙

- 폴더명은 **영문 소문자**만 사용합니다. (예: `fashion`, `real-estate`)
- 폴더 안에 반드시 `page.tsx`가 있어야 URL로 접근 가능합니다.
- 서브 페이지는 폴더 안에 추가 폴더로 구성합니다.

```
templates/fashion/
  page.tsx              ← 메인 페이지 (필수)
  layout.tsx
  cart/
    page.tsx
  category/
    [id]/
      page.tsx
  src/
    components/
    lib/
```

---

## 4. 빌드 시 동작 방식

배포(Vercel) 또는 로컬 개발 서버 시작 시, `templates/` 폴더의 내용이 자동으로 `src/app/`에 복사됩니다.

```
templates/fashion/  →  (자동 복사)  →  src/app/fashion/
```

추가 설정 없이 `/{템플릿명}` URL로 바로 접근 가능해집니다.

관리자 **템플릿 등록** 화면의 **"템플릿연결"** 드롭다운에도 자동으로 나타납니다.

---

## 5. 주의사항 ⚠️

### 절대 수정/삭제하면 안 되는 폴더와 파일

| 경로 | 설명 |
|------|------|
| `src/` | 앱 전체 소스 (templates/ 제외) |
| `scripts/` | 빌드 스크립트 |
| `supabase/` | DB 스키마 |
| `package.json` | 의존성 설정 |
| `vercel.json` | 배포 설정 |

### 그 외 주의사항

- 폴더명이 기존 템플릿과 **중복되면 기존 파일이 덮어씌워집니다.** 반드시 확인하세요.
- 템플릿 내부에서 사용하는 외부 패키지가 있으면 `package.json`에 추가하고 `package-lock.json`까지 함께 커밋하세요.
- 템플릿은 **독립적으로 동작**해야 합니다. `src/lib/`, `src/components/` 등 공통 경로를 import하지 마세요.
- 푸시 전 `npm run build`로 빌드 오류가 없는지 반드시 확인하세요.
