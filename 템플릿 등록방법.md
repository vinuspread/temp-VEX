# 템플릿 등록 방법 (AI 개발자용)

## 1. 템플릿 파일 위치

템플릿 파일은 **`public/templates/static/` 폴더** 안에 폴더 단위로 추가합니다.

```
public/templates/static/
  airline/      ← 기존 템플릿
  car/
  cosmetic/
  my-new-template/  ← 새 템플릿은 여기에 추가
```

> `src/` 폴더는 절대 건드리지 마세요.

---

## 2. 폴더 구조 규칙

- 폴더명은 **영문 소문자**만 사용합니다. (예: `airline`, `real-estate`)
- 폴더 안에 반드시 `index.html`이 있어야 합니다.
- 서브 페이지는 같은 폴더 안에 HTML 파일로 추가합니다.

```
public/templates/static/airline/
  index.html        ← 메인 페이지 (필수)
  about.html
  destinations.html
  fleet.html
  style.css
  main.js
  images/
```

---

## 3. 미리보기 URL

파일을 추가하면 별도 설정 없이 아래 URL로 바로 접근 가능합니다.

```
/templates/static/{폴더명}/
예) /templates/static/airline/
```

관리자 **템플릿 등록** 화면의 **"템플릿연결"** 드롭다운에도 자동으로 나타납니다.

---

## 4. 푸시 방법

```bash
git add public/templates/static/{템플릿명}/
git commit -m "feat: add {템플릿명} template"
git push origin main
```

---

## 5. 주의사항 ⚠️

### 절대 수정/삭제하면 안 되는 폴더와 파일

| 경로 | 설명 |
|------|------|
| `src/` | 앱 전체 소스 |
| `supabase/` | DB 스키마 |
| `package.json` | 의존성 설정 |
| `vercel.json` | 배포 설정 |

### 그 외 주의사항

- 폴더명이 기존 템플릿과 **중복되면 기존 파일이 덮어씌워집니다.** 반드시 확인하세요.
- 템플릿은 **독립적인 정적 HTML**로 작성합니다. Next.js나 React를 사용하지 않습니다.
- 외부 라이브러리는 CDN 링크를 사용하거나 `public/templates/static/{slug}/` 내에 직접 포함하세요.
- CSS, JS, 이미지 등 모든 에셋의 경로는 **상대 경로**로 작성하세요.
